# eBay Seller Connection Setup

Step 13 adds the secure seller OAuth foundation used by later publishing steps. An organization owner or admin can connect one eBay seller account, inspect its connection status, reconnect it, and disconnect it. Listing publication is intentionally not part of this step.

## 1. Configure the eBay RuName

Open the production keyset in the eBay Developer Program and open its **User Tokens** or **Sign in with eBay** settings. Create or edit the production RuName.

Use the public Railway API callback as both the authorization accepted and declined URL:

```text
https://YOUR-API-DOMAIN.up.railway.app/api/ebay/oauth/callback
```

The value saved as `EBAY_RUNAME` is the RuName identifier generated by eBay, not the callback URL. The redirect URI sent during token exchange must exactly match this RuName.

Configure the privacy-policy URL and the other required account-deletion URLs in the eBay portal before saving the RuName.

## 2. Add Railway API variables

Add these variables to the **API service**, alongside the production `EBAY_CLIENT_ID` and `EBAY_CLIENT_SECRET`:

```env
EBAY_ENVIRONMENT=production
EBAY_RUNAME=your-production-runame
EBAY_OAUTH_ENCRYPTION_KEY=one-base64-encoded-32-byte-key
EBAY_OAUTH_SCOPES=https://api.ebay.com/oauth/api_scope/sell.inventory https://api.ebay.com/oauth/api_scope/sell.account https://api.ebay.com/oauth/api_scope/commerce.identity.readonly
WEB_ORIGIN=https://YOUR-WEB-DOMAIN.up.railway.app
```

Generate the encryption key once:

```powershell
node -e "console.log(require('node:crypto').randomBytes(32).toString('base64'))"
```

Keep this key private and stable. Changing or losing it makes existing encrypted seller tokens unreadable; affected organizations must reconnect their eBay accounts. Never commit the value to Git.

After saving the variables, redeploy the API service and verify `/health/ready` returns HTTP 200.

## 3. Connect the seller

1. Sign in to PartPulse as an organization owner or admin.
2. Open the catalog and select **Connect eBay**.
3. Sign in to the intended production seller account on eBay and approve access.
4. eBay redirects to the API callback, which validates a one-use CSRF state, exchanges the authorization code server-side, and redirects to the catalog.
5. Confirm that the catalog header shows the connected account.

Access and refresh tokens are encrypted with AES-256-GCM before storage. Token values are never returned by the connection API or exposed to browser JavaScript. Access tokens are refreshed automatically shortly before expiry.

## API endpoints

- `GET /api/ebay/connection` returns the current organization's sanitized connection status.
- `POST /api/ebay/connection/authorize` creates a short-lived eBay consent URL; owner or admin only.
- `GET /api/ebay/oauth/callback` handles eBay approval or decline and consumes the one-use state.
- `DELETE /api/ebay/connection` removes locally stored token material; owner or admin only.

Disconnecting deletes the local token material but does not promise remote revocation at eBay. A seller can also revoke the application's authorization from their eBay account. Reconnecting replaces the organization's previous seller connection.

## Troubleshooting

- **Seller OAuth is not configured:** one or more of the client ID, client secret, RuName, or encryption key is missing from the API service.
- **Invalid or expired state:** restart from **Connect eBay**. Consent links expire after 10 minutes and can be used only once.
- **Token request failed:** verify production credentials are paired with `EBAY_ENVIRONMENT=production` and that `EBAY_RUNAME` exactly matches the production RuName.
- **Authorization expired:** reconnect the seller. eBay can revoke a grant before its displayed refresh-token expiry.
- **Callback opens the wrong service:** the accepted URL must use the API Railway domain, while `WEB_ORIGIN` must use the web Railway domain.
